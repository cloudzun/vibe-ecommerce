# Phase 4 Context Document — User Authentication

**Date**: 2026-03-05  
**Phase**: 4 of 6  
**Status**: 🟡 In Progress

---

## Project Background

vibe-ecommerce is now a full-stack app (Phase 3): Vercel frontend + Express/SQLite backend at shop-api.huaqloud.com. Products load from API, orders are persisted in database. However, there is no concept of a "user" — all orders are anonymous, cart is client-only localStorage, and all API endpoints are public.

Phase 4 introduces user identity: register, login, JWT-based session, and personal order history.

## What Was Tried Before

- Phase 1-2: Pure frontend, no auth concept
- Phase 3: Introduced backend but deliberately deferred auth to keep scope small
- orders table already has `user_id` column placeholder (NULL = guest order)

## Definition of Done / Failure

**Success**:
- User can register with email + password
- User can log in, receive JWT access token + refresh token
- Logged-in user's orders are associated with their account
- User can view order history at `#account`
- Header shows username when logged in, login link when not
- All auth endpoints protected against brute force (rate limiting)
- JWT secret never in code — environment variable only

**Failure lines**:
- Must not break guest checkout (user_id = NULL still valid)
- Must not break existing cart (localStorage) for guests
- Passwords must never appear in logs or API responses
- Existing Phase 3 API endpoints (/api/products, /api/orders) must remain functional
- No regression on product listing, detail, cart, checkout flows

## Who Is Affected

**Backend new files**:
- `server/routes/auth.js` — register/login/refresh/logout
- `server/middleware/auth.js` — JWT verification middleware
- `server/routes/users.js` — GET /api/users/me/orders

**Backend modified**:
- `server/db.js` — add users table, add user_id to orders
- `server/app.js` — mount new routes, add rate limiter
- `server/ecosystem.config.js` — pm2 env config with JWT_SECRET

**Frontend new files**:
- `js/auth.js` — token storage/refresh/clear utilities
- `js/components/login.js` — #login page
- `js/components/register.js` — #register page
- `js/components/account.js` — #account page (order history)

**Frontend modified**:
- `js/components/header.js` — show username/logout when logged in
- `js/components/checkout.js` — attach user_id if logged in, pre-fill name/email
- `index.html` — add new script tags

**Not touched**:
- router.js, store.js, utils.js, cart.js, products.js, product-detail.js
- All CSS (auth pages reuse existing form styles)

## Constraints

- Vanilla JS frontend — no framework, no build tools
- JWT: access token 15min (localStorage), refresh token 7 days (httpOnly cookie)
- bcrypt cost factor 12
- Rate limit: 10 requests/15min on /api/auth/login and /api/auth/register
- JWT_SECRET via environment variable — never hardcoded
- Guest orders (user_id = NULL) must remain valid — backward compatible
- All new innerHTML must use escapeHtml()
